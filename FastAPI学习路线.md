# FastAPI 零基础学习路线

> 适合没学过前后端的人，按阶段递进，每个阶段都有明确目标和练手项目。
> 总时长约 3~4 个月（每天 1~2 小时），Python 基础好可压缩到 2 个月。

---

## 阶段 0：Python 基础（2~4 周）

FastAPI 是 Python 框架，Python 不熟一切免谈。不需要精通，但以下内容必须掌握：

- **基本语法**：变量、条件、循环、函数、列表/字典/集合
- **面向对象**：类、继承、`__init__`，能看懂别人写的类就行
- **类型注解**：`def add(a: int, b: int) -> int`，这是 FastAPI 的核心依赖，务必重点学
- **异步入门**：知道 `async def` / `await` 是什么意思（先会用，原理后面再补）
- **模块与虚拟环境**：`pip`、`venv`（或 `uv`），会安装和管理依赖

✅ **检验标准**：能独立写一个命令行版的记账小程序（增删查改数据，存到 JSON 文件）。

---

## 阶段 1：理解 Web 是怎么回事（3~5 天）

不写代码，先建立概念，否则后面全是懵的：

- 浏览器输入网址后发生了什么（DNS → 请求 → 响应）
- **HTTP 基础**：请求方法（GET/POST/PUT/DELETE）、状态码（200/404/500）、请求头、请求体
- **前端 vs 后端**：前端管页面展示，后端管数据和逻辑，两者通过 **API（接口）** 用 **JSON** 通信
- 什么是 REST API

✅ **检验标准**：能用自己的话说清楚"点击'登录'按钮后，数据是怎么到服务器又回来的"。

---

## 阶段 2：FastAPI 入门（1~2 周）

跟着 [官方中文教程](https://fastapi.tiangolo.com/zh/tutorial/) 走，官方文档质量非常高，是最好的教材：

1. 安装并跑起来第一个 `Hello World`（`fastapi dev main.py`）
2. **路径参数与查询参数**：`/items/{id}`、`/items?skip=0&limit=10`
3. **请求体与 Pydantic 模型**：这是 FastAPI 的灵魂，理解"数据校验自动完成"
4. **自动文档**：打开 `/docs`（Swagger UI），学会在里面直接测试接口
5. 响应模型、状态码、错误处理（`HTTPException`）

✅ **练手项目**：待办事项（Todo）API——增删查改，数据先存内存里的字典即可。

---

## 阶段 3：数据库（1~2 周）

真实应用的数据要持久化：

- **SQL 基础**：SELECT / INSERT / UPDATE / DELETE，用 SQLite 练（零配置）
- **ORM**：学 SQLModel（FastAPI 作者写的，和 FastAPI 无缝衔接）或 SQLAlchemy 2.0
- 表关系：一对多、多对多
- 数据库迁移工具 Alembic（了解即可）

✅ **练手项目**：把阶段 2 的 Todo API 改成 SQLite 存储，重启服务数据不丢。

---

## 阶段 4：进阶核心功能（2~3 周）

- **依赖注入**（`Depends`）：FastAPI 最重要的设计，管数据库连接、权限校验都靠它
- **用户认证**：注册/登录、密码哈希（bcrypt）、JWT Token、OAuth2
- **项目结构**：拆分 routers、models、schemas、crud，告别单文件
- 中间件、CORS（前端跨域访问必须懂）
- 文件上传、后台任务（`BackgroundTasks`）

✅ **练手项目**：多用户博客 API——用户注册登录，只能编辑自己的文章。

---

## 阶段 5：前端最小集（1~2 周，可选但推荐）

不用成为前端工程师，但要能自己验证 API 的完整链路：

- HTML/CSS 基础（能搭个表单页面）
- JavaScript 的 `fetch()` 调用你的 API
- 或者更省事：直接学一点 **Jinja2 模板**（FastAPI 支持服务端渲染）

✅ **检验标准**：给博客 API 做一个能登录、发文章的简陋网页。

---

## 阶段 6：测试与部署（1~2 周）

- **测试**：`pytest` + FastAPI 的 `TestClient`，给核心接口写测试
- **异步深入**：回头理解 `async/await` 原理，什么时候用异步、什么时候用同步
- **部署**：Docker 基础 → 部署到一台云服务器（或 Railway/Render 这类免费平台）
- 环境变量与配置管理（`pydantic-settings`）

✅ **毕业项目**：完整做一个小产品并上线，比如短链接服务、个人书签管理、或带前端页面的博客。

---

## 学习建议

- **以官方文档为主线**，视频教程只做辅助——FastAPI 官方教程本身就是渐进式课程
- **每个阶段必须动手写项目**，只看不写等于没学
- 遇到报错先自己读错误信息，这本身就是后端工程师的核心技能
- 学有余力再看：Redis 缓存、WebSocket、消息队列、CI/CD
