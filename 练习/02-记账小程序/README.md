# 练习：命令行记账小程序

这是阶段 0 的毕业项目。**不要一口气写完**，按下面 5 步走，每步跑通再进入下一步。

`main.py` 里已经搭好了骨架，你只需要填 `# TODO` 的地方。

## 程序长什么样

运行后是一个循环菜单：

```
===== 记账本 =====
1. 记一笔
2. 查看所有记录
3. 删除一条记录
4. 修改一条记录
0. 退出
请选择:
```

数据存到 `records.json`，格式是"列表里装字典"：

```json
[
  {"id": 1, "item": "午饭", "amount": 25.0},
  {"id": 2, "item": "地铁", "amount": 4.0}
]
```

## 分步计划

### 第 1 步：菜单循环（不碰数据）
- 用 `while True` 写一个循环，打印菜单，用 `input()` 读选项
- 选 0 就 `break` 退出，其他选项先随便打印一句"功能开发中"
- ✅ 跑通标准：能反复显示菜单，输 0 能退出

### 第 2 步：记一笔 + 查看（数据先放内存）
- 用一个 `records: list[dict]` 变量存记录
- "记一笔"：`input()` 读入项目名和金额，组装成字典 append 进列表
- "查看"：循环打印每条记录
- ⚠️ 注意：`input()` 读到的永远是字符串，金额要 `float()` 转换
- ✅ 跑通标准：记 3 笔，能看到 3 条记录和总支出

### 第 3 步：存到 JSON 文件（关键一步！）
- 学两个函数：`json.dump()`（写文件）、`json.load()`（读文件）
- 程序启动时：如果 `records.json` 存在就读进来，不存在就用空列表
- 每次增删改之后：整个列表重新写回文件
- ✅ 跑通标准：记几笔 → 退出 → 重新运行 → 记录还在

### 第 4 步：删除和修改
- 每条记录要有 `id`（新纪录的 id = 当前最大 id + 1）
- 删除：让用户输入 id，从列表里找到并移除；找不到要提示
- 修改：输入 id 找到那条记录，重新输入项目名和金额
- ✅ 跑通标准：删掉一条后 id 不会重复，改一条后文件里也变了

### 第 5 步：健壮性（加分项）
- 用户金额输了 "abc" 怎么办？用 `try/except ValueError` 处理
- 删除不存在的 id、文件内容损坏，都不能让程序崩溃
- ✅ 跑通标准：怎么乱输都不会看到报错堆栈

## 卡住了怎么办

1. 先读报错信息的**最后一行**，它会告诉你错误类型和原因
2. 打印中间变量看看数据长什么样（`print(records)`）
