# 阶段 1：理解 Web 是怎么回事

这个阶段不写多少代码，目标是建立**心智模型**。学完后你要能用自己的话回答文末的 4 个问题。

---

## 1. 浏览器输入网址后发生了什么

以访问 `https://example.com/hello` 为例：

```
你按下回车
  ↓
① DNS 解析：把域名 example.com 翻译成服务器的 IP 地址（相当于查电话簿）
  ↓
② 建立连接：浏览器和那台服务器建立 TCP 连接
  ↓
③ 发送 HTTP 请求："我要 GET /hello 这个资源"
  ↓
④ 服务器处理：服务器上的程序（以后就是你写的 FastAPI！）收到请求，
   执行代码、查数据库，组装出响应
  ↓
⑤ 返回 HTTP 响应：状态码 + 数据（HTML 页面或 JSON）
  ↓
⑥ 浏览器渲染：把收到的内容展示出来
```

**核心认知**：你以后写的后端代码，就是第 ④ 步那个"收到请求 → 返回响应"的程序。Web 开发的一切都围绕"请求-响应"这一来一回。

## 2. HTTP 请求和响应长什么样

HTTP 报文就是有固定格式的纯文本。一个请求：

```
POST /records HTTP/1.1          ← 请求行：方法 + 路径
Host: example.com               ← 请求头（headers）：附加信息
Content-Type: application/json  ← 告诉服务器"我发的是 JSON"
                                ← 空行分隔
{"item": "午饭", "amount": 25}  ← 请求体（body）：真正的数据
```

对应的响应：

```
HTTP/1.1 201 Created            ← 状态行：状态码 + 说明
Content-Type: application/json  ← 响应头：告诉浏览器"我返回的是 JSON"

{"id": 3, "item": "午饭", "amount": 25}   ← 响应体
```

### 请求方法（你只需先记 4 个）

| 方法 | 语义 | 类比你的记账程序 |
|---|---|---|
| GET | 读取数据 | 查看所有记录 |
| POST | 新增数据 | 记一笔 |
| PUT | 修改数据 | 修改一条记录 |
| DELETE | 删除数据 | 删除一条记录 |

有没有发现？**和你记账程序的菜单一一对应**。方法只是"约定的语义标签"，真正做什么由服务器代码决定。

### 状态码（先记这 6 个）

| 状态码 | 含义 | 什么时候见到 |
|---|---|---|
| 200 | OK，成功 | 最常见 |
| 201 | Created，创建成功 | POST 新增之后 |
| 401 | 未登录/凭证无效 | 没带 token 就访问需要登录的接口 |
| 403 | 没权限 | 登录了但不许你操作别人的数据 |
| 404 | 找不到 | 路径写错、id 不存在 |
| 500 | 服务器内部错误 | **你的后端代码抛异常了**（以后会常见😄） |

规律：2xx 成功，4xx 是**客户端**的锅，5xx 是**服务器**（你）的锅。

## 3. 前端 vs 后端

- **前端**：跑在**用户的浏览器**里（HTML/CSS/JavaScript），负责界面展示和交互
- **后端**：跑在**服务器**上（你的 Python 程序），负责业务逻辑和数据
- 两者不在同一台机器上，只能通过 **HTTP 请求 + JSON** 隔空对话
- **API（接口）**：后端暴露出来的一组"可访问的网址"，每个网址代表一个功能

上一节你已经知道 JSON 是纯文本格式——现在明白它为什么重要了：它就是前后端之间的"通用语言"。

## 4. 什么是 REST API

REST 是一种**给接口起名的约定**：用"名词路径"表示资源，用"HTTP 方法"表示动作：

```
GET    /records      → 获取所有记录
POST   /records      → 新增一条记录
GET    /records/3    → 获取 id=3 的那条
PUT    /records/3    → 修改 id=3 的那条
DELETE /records/3    → 删除 id=3 的那条
```

不用 `/getRecords`、`/deleteRecordById` 这种动词路径——动作已经由方法表达了。

## 动手实验 1：亲眼看一次 HTTP（5 分钟）

1. 打开浏览器，按 **F12** 打开开发者工具，切到 **网络（Network）** 标签
2. 访问任意网站（比如 https://www.bing.com）
3. 点击列表里第一条请求，观察：
   - 请求方法（GET）和状态码（200）
   - **标头/Headers**：请求头和响应头
   - **响应/Response**：服务器返回的 HTML
4. 随便改 URL 访问一个不存在的路径，看看 404 长什么样

## 动手实验 2：跑一个最原始的后端（20 分钟）

这个目录下有两个文件：

- `mini_server.py`：用 Python 标准库手写的 HTTP 服务器，提供"记账本 API"
- `client.py`：模拟前端，用代码向服务器发请求（有两个 TODO 要你补）

步骤：

1. 打开**第一个终端**，运行服务器：`python mini_server.py`（它会一直运行，别关）
2. 浏览器访问 http://127.0.0.1:8000/records —— 你在浏览器里发了一个 GET 请求！
3. 再访问 http://127.0.0.1:8000/xxx —— 亲手制造一个 404
4. 打开**第二个终端**，运行 `python client.py`，补全里面的 TODO
5. 读一遍 `mini_server.py` 的代码（不要求会写），感受一下：解析路径、设置状态码、
   转换 JSON……全是手工活。**FastAPI 就是把这些手工活全部自动化的工具**——
   阶段 2 你会用 5 行代码实现同样的功能。
