# 阶段 2：FastAPI 入门

环境已就绪：conda 环境 `fastapi_env`（FastAPI 0.138 + uvicorn 已安装）。

## 每次开工的固定动作

在本目录（`练习/04-FastAPI入门`）打开终端：

```powershell
conda activate fastapi_env
fastapi dev hello.py
```

看到 `Uvicorn running on http://127.0.0.1:8000` 就成功了。
**改代码保存后服务器自动重启**，不用手动重开。按 `Ctrl+C` 停止。

## 学习路径（约 1~2 周）

### 第 1 步：跑通 hello.py（第 1 天）

1. 启动后访问 http://127.0.0.1:8000/ 和 **http://127.0.0.1:8000/docs**
2. `/docs` 是自动生成的交互文档——点开任意接口 → "Try it out" → 填参数 → "Execute"，
   能直接发请求看响应。**以后测试接口全用它**，这就是 FastAPI 最惊艳的功能之一
3. 把 hello.py 的 5 节代码从上到下读一遍，每节都在 /docs 里实际试一次，
   特别要试"故意输错"：`/users/abc`、amount 填 `"abc"`，观察 422 错误——
   体会类型注解是怎么变成校验规则的

### 第 2 步：跟官方教程系统过一遍（第 2~5 天）

hello.py 只是开胃菜，正餐是[官方中文教程](https://fastapi.tiangolo.com/zh/tutorial/)。
按顺序读这几章就够（后面的章节阶段 4 再学）：

- [x] 第一步 & 路径参数 & 查询参数（hello.py 已覆盖，快速过）
- [ ] 请求体
- [ ] 查询参数和字符串校验（`Query`，了解即可）
- [ ] 响应模型 `response_model`
- [ ] 处理错误
- [ ] JSON 兼容编码器（了解即可）

方法：每章的示例代码都亲手敲进 hello.py 跑一遍，不要复制粘贴。

### 第 3 步：毕业项目——记账本 API（第 6~10 天）

打开 `记账API.py`：把你阶段 0 写的记账程序改造成 REST API，
路由就是你阶段 1 毕业检验里设计的那 5 个接口。

- 第一个接口我已写好当示例，剩下 4 个 TODO 自己完成
- 每写完一个就去 /docs 测试：正常数据、错误数据、不存在的 id 都要试
- 全通过后做文件底部的 2 个进阶任务（统计接口 + JSON 持久化）
